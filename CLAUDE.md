遵守以下 Git 工作流：环境检查： 请先帮我检查电脑是否安装了 Git。如果没有，请指导我安装，或者直接尝试帮我安装
原子化提交： 每当完成一个小的功能点（比如“写好了导航栏”或“修复了图片不显示的 bug”），不管是大是小，只要代码能跑，就必须主动帮我运行 git 提交命令
自动写日志： 提交信息（Commit Message）根据刚才修改的内容自动生成，提交信息格式符合主流规范，内容要简单易懂（例如：“feat: 新增了用户登录按钮”）
无需确认： 只要代码没报错，就直接帮我提交，不需要每次都问我
后续所有前端代码请严格遵循以下规范：保持 Next.js App Router + React 19 + TypeScript 架构；样式仅用 Tailwind CSS 4.x，优先复用现有 shadcn/ui 与 design token；图标仅用 lucide-react（必要时可用 @iconify/react）；仅在确有需求时使用 echarts + echarts-for-react；必须支持亮/暗主题（系统主题优先 + 手动切换并持久化）；必须完整响应式（桌面优先并兼顾平板/手机）；视觉风格为简约 Apple-like（高留白、弱分割线、统一圆角与间距、克制动效）；不得改变现有业务逻辑和 API 行为；优先复用现有组件与页面结构，避免无必要重写；新增/修改代码全部使用 TypeScript 且保证类型安全。
